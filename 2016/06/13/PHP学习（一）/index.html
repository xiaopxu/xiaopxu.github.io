环境搭建（MAC OS系统）

XAMPP：

集成了Apache+MySQL+PHP+Perl

Apache：

服务器软件，可以运行在几乎所有计算机平台，最流行的服务器软件之一

MySQL：

关系型数据库管理系统，体积小，速度快，开源

安装完毕后需要手动开启服务

可能会遇到以下问题（这个没有测试过，因为我开的时候没问题）

Apache无法开启：

原因：之前已经将系统自带的Apache开启，需要手动关闭

sudo apachectl stop
MySQL无法开启：

原因：之前已经安装过MySQL并默认开启，需手动关闭

sudo/Library/Startupltems/MySQLCOM/MySQLCOM stop

使用ps-eflgrep mysql查找进程，用kill杀掉进程

常量

  • 常量是指在程序执行中无法修改的值。如 PI (3.1415926);
  • 在脚本执行期间该值不能改变;
  • 常量对大小写敏感,通常常量名总是大写;
  • 常量是全局的,可以在脚本的任何地方引用;
  • 常量分为内置常量和自定义常量;
  • 常量使用define()函数定义;

赋值

值赋值
$age = 18;
$age2 = $age;
echo $age2; //18

//此时将$age2的赋值改变，age不会变化
//因为传值是复制一份重新存一个内存地址传给新的变量，所以是指向两个内存地址
$age2 = 20;
echo $age2; //18
引用赋值
//&引用传值符，变为传址，把一个变量的引用赋给另一个变量
$age = 18;
$age2 = &$age;  

//传址情况下，$age与$age2是指向的同一个内存地址
//所以改动一个，另一个会跟着变动
$age2 = 20;
echo $age; //20

数据类型

标量类型

字符串

单引号：出现的变量不会被其值替代

$name = "zhangsan";
$str = '姓名：';
//字符串连接用"."
echo $str.$name;  //姓名：zhangsan

//变量$name不会被其值替换
$name = "zhangsan";
$str = '姓名：$name';
echo $str;  //姓名：$name

双引号：出现的变量会被其值替代

//变量$name被其值替代
$name = "zhangsan";
$str = "姓名：$name";
echo $str;  //姓名：zhangsan

PS1：引号中的变量是否解析看最外层的引号类型决定

//变量能否被解析，取决于最外层是否双引号
//如果是双引号，能解析；如果不是双引号，不能解析

//最外层是双引号，所以解析
$name = "zhangsan";
$str = "姓名：'$name'";
echo $str;  //姓名：'zhangsan'

//最外层是单引号，所以不解析
$name = "zhangsan";
$str = '姓名："$name"';
echo $str;  //姓名："$name"

//最外层是双引号，所以解析
$name = "zhangsan";
$str = "姓名：'$name is a boy'";
echo $str;  //姓名：'zhangsan is a boy'

PS2：变量后的内容需要空格分开，不然会有undefined报错，可用{}隔断

$name = "zhangsan";
$str = "姓名：'$nameis a boy'";
echo $str;  //报错nameis未定义

//花括号，阻断
// $name = "zhangsan";
// $str = "姓名：'{$name}is a boy'";
// echo $str;  //姓名：'zhangsanis a boy'

//界定符：区间内容原样输出
    $artical = <<<EOD

自动转换

php对于类型定义非常松散，所以会根据引用变量的环境，将变量自动转换为最合适的类型

$e = "abc123";
echo $a + $b + $c + $d + $e; //10+1+23+1.2  35.2
