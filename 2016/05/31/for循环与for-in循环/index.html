JavaScript中对象的循环方法--for循环与for-in循环

  • JavaScript

在JavaScript中如果想要对一个对象进行循环，可以使用for或者for-in两种语句；让我们来看看这两种语句在对象循环上有些什么区别

for语句

for语句对于数组的循环比较适用

//基本的数组可以循环
var arr = [1,2,3,4]
for (var i = 0; i<arr.length; i++){
    console.log(arr[i]);    //1 2 3 4  
}

//复杂的数组也可以循环
var arr = [1,2,[3,4],{a:'abc',b:true}]
for (var i = 0; i<arr.length; i++){
    console.log(arr[i])    //1 2 [3,4] {a:'abc',b:true}  
}

如果是对一个对象循环就会出现问题

//但是，如果是一个对象，就会出现问题
var obj = {a:123, b:'abc', c:true}
for (var i = 0; i<obj.length; i++){
    console.log(obj[i]);    //在chrome浏览器下调试，不出现任何东西，也没有报警
}
原因：

  • 问题出在语句的判断条件i < obj.length，obj.length是undefined

  • 所以现在就是在判断i < undefined，也就是判断i < NAN，所以结果一定是false

  • 因此语句根本不会被执行，所以不会有任何东西被打印出来

但是对象的情况下也有特例：
var obj = {
    '0' = 123,
    '1' = 'abc',
    '2' = true,
    length = 3
};
for (var i = 0; i<obj.length; i++){
    console.log(i)    //0 1 2
    console.log(obj[i]);    //123 'abc' true
}

给obj定义一个长度，并给每个元素定义一个位置下标（而不是使用abc，寻找不到的位置下标会打印undefined），就可以正常使用for语句

var obj = {
    'a' = 123,
    '1' = 'abc',
    '2' = true,
    length = 3
};
for (var i = 0; i<obj.length; i++){
    console.log(i)    //0 1 2  可以看到第一个位置还是用0坐标去获取的
    console.log(obj[i]);    //undefined 'abc' true  第一个的位置下标不是0，导致打印undefined
}

for-in语句

for-in语句可以用来枚举对象属性，每次循环时，都会将obj中存在的一个属性名赋值给变量i，直到属性全部被枚举一遍

var obj = {a:123,b:'abc',c:true};
for (var i in obj){
    console.log(i)    //a b c
    console.log(obj[i]);    //123 'abc' true
}

for-in语句还可以枚举出document等对象的所有属性

var n = 0;
for (var i in document){
    n++
    console.log(document[i])  //一大堆属性
}
console.log(n);  //在chrome下232（这个数可能每个浏览器会有差别）

以下这点需要特别注意一下

ECMAScript对象的属性没有顺序，因此通过for-in循环输出的属性名顺序是不可测的。具体来讲，所有属性都会被返回一次，但返回的先后次序可能会因浏览器而异 —-《JavaScript高级程序设计》
